# PRD - 讀書筆記系統

## 1. 專案概述

### 背景與動機
在資訊爆炸的時代，人們閱讀書籍後往往容易忘記內容與心得。為了解決這個問題，並幫助讀者建立個人專屬的知識庫，我們希望能打造一個「讀書筆記系統」，讓使用者可以輕鬆記錄、回顧，並搜尋過去閱讀過的書籍資訊、心得與評價。

### 目標用戶
- 讀書愛好者
- 需要整理學習筆記的學生或專業人士
- 喜歡記錄生活點滴與閱讀心得的人

### 核心價值主張
提供一個簡潔、直覺的平台，讓使用者能快速記錄每一本讀過的書，包含書名、心得與評分，並能透過搜尋功能快速找回過去的閱讀記憶，甚至透過系統推薦獲得下一本好書的靈感。

---

## 2. 功能需求

### 主要功能與使用者故事

1. **紀錄書籍資訊 (紀錄)**
   - 作為 **讀書愛好者**，我希望 **能新增書籍的基本資訊（如書名、作者等）**，以便 **將我讀過的書建檔管理**。

2. **書籍評分 (評分)**
   - 作為 **讀書愛好者**，我希望 **能對讀完的書籍給予 1 到 5 星的評分**，以便 **快速了解自己對這本書的喜好程度**。

3. **撰寫閱讀心得 (心得)**
   - 作為 **讀書愛好者**，我希望 **能自由撰寫並編輯閱讀心得或筆記**，以便 **記錄下閱讀時的感悟與重點**。

4. **搜尋書籍與筆記 (查詢)**
   - 作為 **讀書愛好者**，我希望 **能透過關鍵字（如書名、作者、心得內容）搜尋書單**，以便 **在需要時快速找回特定的閱讀紀錄**。

5. **系統推薦 (推薦資料)**
   - 作為 **讀書愛好者**，我希望 **系統能根據我的閱讀紀錄或高分評價，推薦相關的書籍或資料**，以便 **探索更多感興趣的閱讀內容**。

---

## 3. 非功能需求

### 技術限制
- 後端框架：Flask (Python)
- 前端技術：HTML / Vanilla CSS / Jinja2 模板引擎
- 資料庫：SQLite

### 效能與安全考量
- **效能**：系統回應時間應小於 1 秒（特別是搜尋功能）。
- **安全**：
  - 防止 SQL Injection 與 XSS 攻擊。
  - 確保使用者的個人筆記資料隱私不被未經授權的存取（若未來有帳號系統）。
